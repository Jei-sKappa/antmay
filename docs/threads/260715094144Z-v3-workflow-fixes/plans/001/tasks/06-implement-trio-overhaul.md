# Task 6: Overhaul the implement trio

**Objective:** Bring `implement`, `implement-plan`, and `implement-plan-with-subagents` onto the unified blocked protocol, the bounded failed-commit loop, extracted roadmap feedback, and persistent `<UTC>[-<desc>]` run workspaces.

**Input / context:** Settled decisions per `seed/discussions/260715102305Z-v3-skill-review-notes-decision-log.md` P1 (unified protocol), P2 (failed-commit loop), P11 (roadmap-feedback extraction), P14 (no deletion; bodies silent; positive persistence framing), P15 (run-dir naming). The three files: `skills/workflow/implement/implement/SKILL.md`, `skills/workflow/implement/implement-plan/SKILL.md`, `skills/workflow/implement/implement-plan-with-subagents/SKILL.md`. Read all three fully first; they share section structure, so apply each change consistently across the trio.

**Steps:**

1. In each file, replace the blocked section (`## Blocked and AFK` in `implement`; the equivalent in the other two) with a `## Blocked` section per P1: no attended branch, no AFK detection; finish everything the run can safely derive, bundle the indispensable open decision(s) via `/emit-pending-decisions` (producer, target, evidence, the originating user request, suggested follow-up), update the implementation report per its section to reflect the blocked outcome, end with a concise notification naming the bundle. The "record the answer to `decisions.md` before acting" attended clause is deleted (recording happens at bundle resolution, not here).
2. In each file, rewrite `### Failed commit` per P2 as the bounded fix loop:
   - Diagnose first — read the actual error output; never retry blind.
   - Fix in-authority causes as part of the current task: lint/format violations in the task's files, a hook that auto-modified files needing a re-stage, a test broken by the task's own diff, a commit subject rejected by a message linter; re-run the failed check, retry the commit.
   - At most 3 fix-and-retry attempts per task. Past the cap, or when the cause is outside the task's authority (missing sign-off config, credentials, failures in files the task doesn't own, infra errors): report `BLOCKED` with the diagnosis in the notes (not just "commit failed"), append the progress block with `Commit: none` where the skill uses progress blocks, stop the run.
   - Guardrails restated explicitly: never bypass hooks (`--no-verify` or equivalent), never weaken/delete/skip a check to make it pass, never stash-and-retry. `### No history rewriting` stays byte-identical, including its pairing sentence (update only its cross-reference if the failed-commit section's framing sentence changed).
   - Audit trail: the task report and commit body note the commit failed N times and what was fixed.
   - Update the step-list clauses that currently say "If commit fails, report `BLOCKED` … and stop" to route through the rewritten section.
3. Create `shared/references/roadmap-descendant-feedback.md` from the current `## Roadmap-descendant feedback` body text (what qualifies as parent-level impact, what to supply to `/append-roadmap-feedback`, what stays local); add all three implement skills to `shared/manifest.yaml` for it; run `node scripts/sync-shared-references.mjs`. In each SKILL.md, replace the section body with the inline trigger condition plus a naturally-worded pointer to `references/shared/roadmap-descendant-feedback.md` (no mechanical IF/READ phrasing).
4. In each file, apply P14: delete the removal clauses — the "then remove the run workspace / this run's workspace directory" sentence in the step list, the "the directory is removed (see `## Implementation report`)" sentence in `## Run workspace`, and the "After the report is written/updated, remove this run's … directory" sentence in `## Implementation report`. Add one positively-framed sentence in `## Run workspace`: the directory remains in place after the run as the run's operational trace. Do not use delete/remove tokens; add no deletion prohibition either — bodies stay silent. Keep the no-reference rule (no durable artifact references a run-directory path) and adjust the "transient scratch" framing to "invocation-scoped" where it now reads wrong.
5. In each file, apply P15 to `## Run workspace`: the directory is `.implementation-runs/<UTC>[-<desc>]/` — `<UTC>` mandatory and sufficient; `<desc>` an optional short kebab descriptor of the run's objective (from the input's subject for `implement`; from the plan's title for the plan skills). Remove the `plan-` infix in `implement-plan-with-subagents` and every occurrence of `<ref>`, "descriptor", and "`001`" in the naming explanations.
6. Re-read each file end-to-end: no removal language, no `<ref>`, no attended/AFK vocabulary, no inline roadmap-feedback body, failed-commit loop consistent with the step list.
