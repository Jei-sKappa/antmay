# Seed: V3 workflow kinds
External: none — this exploration began as a local idea without a tracker ticket

Explore a possible V3 workflow that replaces numeric tiers with nominal kinds defining a thread's structure. Kinds should support distinct sub-workflows, including a roadmap or umbrella thread that develops a project from scratch and opens implementation-phase threads, a normal thread, and a quick thread that moves from brief discussion to a short plan or directly to implementation.
