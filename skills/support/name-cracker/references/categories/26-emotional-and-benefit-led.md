# Emotional and benefit-led names

Names that either state how the product should make the user feel or state the concrete result the user gets from using it. Both angles skip the mechanism entirely and sell the payoff directly — an internal state on one side, an outcome on the other.

## Techniques

- State the feeling the product should produce (Calm, Joy, Brave)
- Use an abstract emotional or mental-state noun as the whole name (Headspace, Linear)
- Name the concrete outcome the user gets (Fastly for speed, LastPass for "the last password you'll need")
- Pair a value adjective with the product category (SecureWorks, QuickBooks, BetterHelp)
- Compress the benefit directly into the product noun (Evernote)

## Examples

Calm, Headspace, Brave, Linear, Fastly, QuickBooks, Evernote, LastPass, BetterHelp

## Pitfalls

- Feeling-based names can overpromise or read as vague if the product doesn't deliver that feeling.
- Benefit-led names are easy for a competitor to match with an equally plausible claim.
