# Meta and placeholder names

Names that either comment openly on their own artificiality or started life as a temporary label and never got replaced. Both moves turn the absence of a "real" name into the identity itself, and the honesty of that gesture is often what makes it memorable.

## Techniques

- Name that literally describes its own made-up-ness (BAML, expanding to "Basically A Made-up Language")
- Adopt a classic programming placeholder as the permanent name (Foo, Bar, Baz)
- Keep the literal "not yet named" state as the shipped name (Untitled, Nameless)
- Reuse a generic corporate stand-in name (Acme, NewCo)
- Let a temporary internal marker survive into the final product (TBD, Project X)

## Examples

BAML, Foo, Bar, Baz, Untitled, Acme, NewCo, Project X, TBD

## Pitfalls

- Can read as unserious or unfinished, undermining confidence in the product.
- Carries weak semantic differentiation — the name signals only that it was never properly chosen.
