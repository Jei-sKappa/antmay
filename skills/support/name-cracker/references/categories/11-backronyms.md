# Backronyms and recursive acronyms

Pick the short, pronounceable word first, then build — or retrofit — an expanded phrase whose initials justify it. A recursive variant folds the acronym back into its own expansion, so the name refers to itself. Both combine memorability with a wink at the audience.

## Techniques

- Choose a catchy short word first, then reverse-engineer a phrase whose initials match it (RPM → RPM Package Manager, JACK → JACK Audio Connection Kit)
- Make the expansion self-referential so the acronym reappears inside its own definition (GNU → GNU's Not Unix, YAML → YAML Ain't Markup Language, WINE → Wine Is Not an Emulator)
- Lean into technical humor or an in-joke shared with the target community (BAML → Basically A Made-up Language)
- Retrofit a formal expansion onto an already-popular informal short name (PHP → PHP: Hypertext Preprocessor)

## Examples

GNU, YAML, WINE, BAML, RPM, JACK, PHP

## Pitfalls

- The expansion can sound contrived or visibly reverse-engineered rather than natural.
- Recursive forms confuse audiences unfamiliar with the self-referential convention.
