# Metaphorical names

The name draws an analogy between the product's function and an unrelated concrete domain, letting borrowed imagery carry meaning the literal name never states outright.

## Techniques

- Borrow from navigation or piloting for orientation and control tools (Kubernetes → helmsman)
- Borrow from construction or reshaping for tools that build or transform an environment (Terraform)
- Borrow from shipping and logistics for tools that package and move things (Docker → shipping containers)
- Borrow from a controlled workspace for collaborative or experimental tools (GitLab → laboratory for Git work)
- Borrow from fiction for a specific evocative property (Ansible → instantaneous communication device)

## Examples

Docker, Kubernetes, GitLab, Terraform, Ansible, Apache Kafka

## Pitfalls

- The metaphor can be obscure or misleading to anyone unfamiliar with its source domain.
- A rich metaphor invites overly literal expectations about what the product actually does.
