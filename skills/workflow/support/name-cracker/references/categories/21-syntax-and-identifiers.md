# Syntax and identifier names

The name is styled after programming syntax — a file extension, a shell command, a filesystem path — or after a code identifier such as a variable, package, or namespace, signaling developer orientation at a glance.

## Techniques

- Style the name as a file extension or filename (`.NET`, Node.js, Vue.js, README)
- Style the name as a filesystem path or protocol notation (`/dev/null`, localhost)
- Style the name as a lowercase package or library identifier (lodash, stdlib, pytest, systemd)
- Apply a code casing convention to the brand itself: camelCase, PascalCase, snake_case, kebab-case (TypeScript, Next.js, OpenCV)
- Use a namespace-style prefix to signal a family or ecosystem (XCTest)

## Examples

Node.js, Deno, Vue.js, .NET, npm, README, lodash, stdlib, TypeScript, Next.js, systemd

## Pitfalls

- Punctuation and unusual casing can break search engines, voice assistants, and trademark filings.
- The style locks branding into looking "for developers," which can limit appeal to a broader audience.
