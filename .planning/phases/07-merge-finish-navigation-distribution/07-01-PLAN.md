Ship the V1 MERGE pair: `merge-artifacts-auto` and `merge-artifacts-interactive` — the cross-cutting merge skills that reconcile two or more V1 artifacts into a NEW next-mainline-integer version of the target type. Same-type merge is the default (e.g., two specs -> one spec). Cross-type merge is allowed when the target type is EXPLICITLY stated by the user or obvious from context (e.g., 'merge proposal + discussion into a spec' -> output lands in `specs/`). The interactive variant is an explicit EXCEPTION to D93's 'interactive may or may not log' default — merge interactions ARE the durable trade-offs and the decision log is mandatory.
