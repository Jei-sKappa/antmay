<objective>
Ship the granularity-shifting pair: `adjust-plan-granularity-auto` (autonomous: existing plan + target instruction → new versioned plan) and `adjust-plan-granularity-interactive` (collaborative walk: per-task split/merge/expand/contract/leave decisions → new versioned plan). Both skills emit a NEW versioned plan at `docs/threads/<thread>/plans/<UTC>-v<N+1>[-<descriptor>]-plan.md` where the descriptor encodes the granularity shift (e.g., `v2-stricter-plan.md`, `v2-impl-ready-plan.md`). The original source plan stays IMMUTABLE per D39. Both skills enforce the D59 sequential-isolated-independent contract, the D60 strict prohibition on parallelization markers, the D61 self-review pass before emission, and the D62 never-auto-commit rule. Register both skills in the three V1 touchpoints.

Purpose: The granularity-shifting pair is the third of three pairs in Phase 4. It lets a user adapt an existing plan when they started with the wrong granularity or need to retarget for a different implementer — without losing the original version. Per D57, the skill interprets degree naturally rather than relying on a fixed list of transformations.

Output: Two new skill folders + three registration files updated. After this plan, all 6 new Phase 4 skills are installable. The JeisKappa-workflow plugin reaches its final Phase 4 count of 13 entries.
