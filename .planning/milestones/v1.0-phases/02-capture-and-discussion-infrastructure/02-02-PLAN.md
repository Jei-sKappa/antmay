<objective>
Ship the two open-ended-and-seeded discussion skills that together replace the behavioral surface of the legacy `discussion-loop`: `discussion` (open-ended interview; questions discovered live; options + recommendation OPT-IN per decision point) and `seeded-discussion` (predetermined point walk; one point at a time; options + recommendation default-on; reuses the legacy Loop section verbatim). Both write append-only decision logs to `docs/threads/<thread>/discussions/<UTC>-<kebab-desc>-decision-log.md` per the Phase 1 record-form grammar (decision-log token per D94 "the artifact is a decision log; skill names keep 'discussion'"). Both register through all four CLAUDE.md "When adding a new skill" touchpoints.

Purpose: Without these two skills, V1 has no way to drive a decision-making conversation that produces a sequentially-numbered, append-only, thread-local decision log — every downstream Phase 3+ skill that wants to reference `## D<N>` decisions from a settled discussion (e.g., `spec-*` per DISC-05) would have no source artifact in the V1 thread layout. Keeping both skills in ONE plan is deliberate: ~80% of their bodies (anti-sycophancy stance, decision log filename grammar, append-only logging, scope-drift handling, thread resolution, question budget) is shared — splitting would force the executor to read identical context twice and produces two skills with subtly divergent body voice. The differences (open-ended vs. seeded; opt-in vs. default-on options+recommendation) live in tight, named sections of each skill body and are independently verifiable.

Output: Two new skill folders (`skills/discussion/SKILL.md`, `skills/seeded-discussion/SKILL.md`); three edits to existing registration files. No reference docs.

Depends on Plan 02-01 (`capture-inbox`) because both skills' scope-drift handling sections instruct the agent to park out-of-scope branches via `capture-inbox` (preferred option per CONTEXT.md). Also serializes on `.claude-plugin/marketplace.json` + `.vscode/settings.json` + `README.md` edits — three files Plan 02-01 also touched.
</objective>
